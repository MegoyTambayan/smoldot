pub fn has_output(&self) -> bool

Returns true if the state machine is in a ready state, meaning that it has an “output” finalized block and an “output” best block. In other words, returns true if calling DownloadTree::try_advance_output has returned OutputUpdate::FirstFinalized in the past.

Several methods panic if the state machine isn’t in a ready state.

pub fn input_finalize(
    &mut self,
    hash_to_finalize: [u8; 32],
    new_best_block_hash: [u8; 32]
)

Updates the state machine to take into account that the input of blocks has finalized the given block.

new_best_block_hash is the hash of the best block after the finalization.

Note: Finalizing a block might have to modify the current best block if the block being finalized isn’t an ancestor of the current best block.

Panic

Panics if hash_to_finalize or new_best_block_hash weren’t inserted before. Panics if trying to finalize the parent of a block that is already finalized. Panics if new_best_block_hash is not a descendant of hash_to_finalize.

pub fn try_advance_output(&mut self) -> Option<OutputUpdate<'_>>

Tries to update the output blocks to follow the input.

Should be called after inserting a new block, finalizing a block, or when a download is finished.

Returns None if the state machine doesn’t have any output yet.

If Some is returned, this method should be called repeatedly until OutputUpdate::None is returned. Each call can perform an additional update.
